# 暮らしAIアシスタント — AI相談UX設計書

*作成日: 2026-06-28*

---

## 設計方針

### このサービスが提供したいこと

> 「検索して調べる」ではなく、「話したら答えが出てくる」体験

役所の窓口に来た人に、担当者が丁寧に聞いてくれる感覚。
制度の名前を知らなくても、状況を話すだけで「あなたのことを考えて選んでくれた」と思える体験。

---

### UXの5原則

```
1. 聞きすぎない
   質問は最大5問。それ以上は離脱する。
   「あとで詳しく」より「今すぐ3件」のほうが価値がある。

2. 理由を先に言う
   「あなたが失業中であれば〜」ではなく
   「育休中で収入が減っている状況なら、この制度が当てはまります」。
   "なぜ自分に関係あるか"を先に見せる。

3. 専門用語を使わない
   「雇用保険」ではなく「仕事を辞めたときにもらえるお金の制度」。
   制度名は後から出す。

4. ネガティブな状況を責めない
   「生活費が足りない」「失業した」は恥ずかしい状況ではない。
   AIは淡々と、でも温かく聞く。

5. 次の行動を必ず示す
   制度カードを出したら終わりではなく、
   「次は窓口に電話する」「書類を準備する」まで案内する。
```

---

### 会話の共通構造

```
【フェーズ1】入口（0〜1ターン）
  ユーザーが状況を話す
  AIが共感しつつカテゴリを確認する

【フェーズ2】ヒアリング（2〜5ターン）
  AIが3〜5問質問する
  クイックリプライで答えやすく
  フリーテキストも受け付ける

【フェーズ3】提示（1ターン）
  「○件の制度が使える可能性があります」
  優先度順に最大6件のカードを表示
  各カードに「なぜあなたに当てはまるか」の理由を付ける

【フェーズ4】深掘り（任意）
  「もっと詳しく知りたい制度はありますか？」
  「他の状況についても相談しますか？」
  次のカテゴリへの誘導
```

---

### 制度提示の優先度ロジック

```
スコアリング（100点満点）:
  ・条件一致度   40点（年齢・家族・就労が合致するか）
  ・金額の大きさ 20点（給付額が大きいほど上位）
  ・申請のしやすさ 15点（難易度が低いほど上位）
  ・時間的緊急度 15点（締切が近い・今すぐ申請できる）
  ・認知度の低さ 10点（知られていない穴場制度を優先提示）
```

## 01. 子育て

### 入口

**トリガーとなるユーザーの言葉**
```
「子どもが生まれました」「保育園に入れたい」「育休中です」
「児童手当について知りたい」「子育て支援って何がありますか」
```

**AIの第一声**
```
おめでとうございます！（または、大変な時期ですね）
子育てに関する支援は種類が多いので、
状況に合わせて整理しますね。

いくつか確認させてください。
```

---

### 質問フロー

```
Q1「お子さんは今何歳ですか？（または、何ヶ月ですか？）」
   判断目的: 年齢で受給可能な制度が大きく変わる

   └─ 0〜2歳（乳幼児期）
   │     → Q2A（就労確認）へ
   └─ 3〜5歳（保育・幼稚園）
   │     → Q2B（保育園利用確認）へ
   └─ 6〜15歳（小中学生）
   │     → Q2C（収入確認）へ
   └─ 16歳以上・高校生
         → Q2D（高校種別確認）へ

Q2A「現在、お仕事はされていますか？」（乳幼児の場合）
   └─ 育児休業中
   │     → Q3A（雇用保険確認）へ
   └─ 働いていない（専業主婦・夫）
   │     → Q3B（保育希望確認）へ
   └─ 働いている（復職済み）
         → Q3C（保育料確認）へ

Q3A「育児休業前は、会社員・公務員でしたか？」（雇用保険確認）
   └─ はい → 【育児休業給付金】が最優先
   └─ 自営業・フリーランス → 対象外を説明 + 代替制度へ

Q2B「保育園・認定こども園は利用していますか？」（3〜5歳）
   └─ 利用中
   │     →「保育料の無償化が適用されているか確認しましょう」
   └─ 利用していない（幼稚園）
   │     → 幼稚園無償化の説明へ
   └─ どちらも利用していない
         → Q3（保育必要性の確認）へ

Q2C「世帯の年収はおおよそどのくらいですか？」（小中学生）
   └─ 300万円未満 → 就学援助 + 児童手当 を優先
   └─ 300〜600万円 → 児童手当 + 教育費支援
   └─ 600万円以上 → 児童手当（所得制限なし）+ 教育費情報

Q2D「高校の種別を教えてください」（高校生）
   └─ 公立高校 → 高等学校等就学支援金（無償）
   └─ 私立高校 → 同上 + 加算額の説明
   └─ 通信制・定時制 → 対応する制度を案内

最後の確認Q「ひとり親家庭ですか？」（全ルートで聞く）
   └─ はい → 児童扶養手当・ひとり親向け制度を追加
   └─ いいえ → スキップ
```

---

### 提示する制度（優先順）

| 優先 | 制度名 | 提示条件 |
|---|---|---|
| 🥇 | 育児休業給付金 | 育休中 × 雇用保険加入 |
| 🥇 | 幼児教育・保育の無償化 | 3〜5歳の子がいる |
| 🥈 | 出産育児一時金 | 出産後2年以内 |
| 🥈 | 児童手当 | 中学生以下の子がいる（全員） |
| 🥉 | 産前産後休業中の社会保険料免除 | 産休・育休中の会社員 |
| 🥉 | 就学援助制度 | 小中学生 × 低所得 |
| ＋ | 児童扶養手当 | ひとり親の場合のみ追加 |
| ＋ | 特別児童扶養手当 | 子どもに障害がある場合 |

---

### 締めのメッセージ

```
「子育て支援は申請しないと受け取れないものが多いです。
特に児童手当は、出生届の後すぐに市区町村窓口への申請が必要です。

他にも、住んでいる自治体独自の支援があることが多いので、
お住まいの市区町村のサイトも確認してみてください。

次は何について相談しますか？」
```

## 02. 結婚

### 入口

```
「結婚しました」「入籍しました」「これから結婚します」
「結婚を機に引っ越します」
```

**AIの第一声**
```
ご結婚おめでとうございます！
結婚に合わせて使える支援がいくつかあります。
状況を確認させてください。
```

---

### 質問フロー

```
Q1「お二人とも39歳以下ですか？」
   判断目的: 結婚新生活支援事業の年齢要件（39歳以下）
   └─ はい → Q2へ
   └─ いいえ → 年齢制限外の制度のみ案内

Q2「引っ越しや住宅の取得を予定していますか？」
   └─ 新居への引っ越しを予定 → Q3へ
   └─ 住宅購入を予定 → 「住宅購入」カテゴリへ誘導（兼ねる場合も）
   └─ 引っ越しや購入の予定なし → 限定的な制度のみ案内

Q3「世帯年収（お二人合わせて）はどのくらいですか？」
   └─ 500万円未満 → 結婚新生活支援事業の対象可能性が高い
   └─ 500万円以上 → 対象外の可能性 + 他の支援を案内

Q4「引っ越し先は地方（東京圏以外）ですか？」
   └─ 地方・地方移住を考えている
   │     → 移住支援金（最大100万円）を追加
   └─ 都市部内での引っ越し
         → 自治体独自の支援を案内
```

---

### 提示する制度

| 優先 | 制度名 | 提示条件 |
|---|---|---|
| 🥇 | 結婚新生活支援事業（自治体） | 39歳以下 × 収入500万未満 × 引越しあり |
| 🥈 | 移住支援金 | 地方移住予定 |
| 🥉 | 住宅ローン減税 | 住宅購入の場合 |
| ＋ | 健康保険の扶養手続き | 一方が無収入になる場合（制度案内） |

## 03. 妊娠

### 入口

```
「妊娠しました」「妊娠中です」「もうすぐ出産です」
「妊婦健診の費用が気になる」
```

**AIの第一声**
```
ご妊娠おめでとうございます。
出産に向けて使える支援をまとめますね。
いくつか確認させてください。
```

---

### 質問フロー

```
Q1「現在、お仕事はされていますか？」
   └─ 会社員・公務員 → Q2A（育休取得予定を確認）
   └─ 自営業・フリーランス → Q2B（国保確認）
   └─ 専業主婦（夫） → Q2C（配偶者の雇用保険確認）
   └─ 現在無職 → Q2D（状況確認）

Q2A「出産後、育児休業を取得する予定ですか？」
   └─ はい → 育休給付金 + 社会保険料免除を案内
   └─ 退職予定 → 産前産後は働いていた期間で給付金が出る場合あり

Q2B「国民健康保険に加入していますか？」
   └─ はい → 出産育児一時金（国保経由）の案内
   └─ 配偶者の扶養 → 配偶者の健保から出産育児一時金が出る

最終確認「ひとり親（未婚・離別）での出産ですか？」
   └─ はい → 母子家庭向け支援 + 妊娠・出産の特別給付金を案内
   └─ いいえ → スキップ
```

---

### 提示する制度

| 優先 | 制度名 | 提示条件 |
|---|---|---|
| 🥇 | 出産育児一時金（50万円） | 健康保険加入（ほぼ全員） |
| 🥇 | 妊婦健診費用補助（自治体） | 全員（自治体によって14回まで無料） |
| 🥈 | 育児休業給付金 | 会社員 × 育休取得予定 |
| 🥈 | 産前産後休業中の社会保険料免除 | 会社員 × 産休予定 |
| 🥉 | 出産・子育て応援給付金（10万円） | 全員（自治体による） |
| ＋ | 児童手当（出生後の手続き案内） | 出産後すぐに申請が必要と説明 |

## 04. 住宅購入

### 入口

```
「家を買いたい」「マイホームを検討中」「住宅ローンを組む予定」
「新築を買います」「中古住宅のリノベーションを考えている」
```

**AIの第一声**
```
住宅購入には、知らないと損する支援制度がいくつかあります。
購入のタイプによって変わるので確認させてください。
```

---

### 質問フロー

```
Q1「新築・中古・どちらを購入予定ですか？」
   └─ 新築一戸建て・マンション → Q2へ
   └─ 中古住宅（リノベあり） → Q2＋リノベ補助金へ
   └─ 中古住宅（そのまま） → Q2（住宅ローン減税は適用条件が異なる）

Q2「省エネ性能にこだわる予定はありますか？（ZEH・高断熱等）」
   └─ はい / 検討中 → こどもエコすまい支援・ZEH補助金を優先案内
   └─ わからない → 説明してから選択肢を提示
   └─ いいえ → 基本的な住宅ローン減税のみ

Q3「子どもはいますか、または予定がありますか？」
   └─ はい（0〜17歳） → 子育てエコホーム支援事業（最大100万円）対象
   └─ 予定あり → 同上
   └─ いいえ → 一般的な制度のみ案内

Q4「住宅の購入価格（概算）はどのくらいですか？」
   └─ 5000万円以下 → 住宅ローン減税の恩恵が大きい
   └─ 5000万〜1億 → 上限が変わることを説明
   └─ まだ決まっていない → 計算の考え方を案内
```

---

### 提示する制度

| 優先 | 制度名 | 提示条件 |
|---|---|---|
| 🥇 | 住宅ローン減税（最大455万円） | 住宅ローン利用 × 新築または中古 |
| 🥇 | 子育てエコホーム支援事業 | 子あり × 省エネ住宅 |
| 🥈 | こどもみらい住宅支援事業 / ZEH補助金 | 省エネ新築 |
| 🥈 | 住宅取得等資金の贈与税非課税（親からの援助） | 親から資金援助あり |
| 🥉 | フラット35（金利優遇） | 条件確認が必要 |
| ＋ | 不動産取得税の軽減措置 | 新築・中古問わず全員 |

## 05. 引っ越し

### 入口

```
「引っ越しを考えている」「地方に移住したい」「東京から出たい」
「別の市区町村に引っ越します」
```

**AIの第一声**
```
引っ越しの目的や行き先によって、
もらえる支援が大きく変わります。
少し確認させてください。
```

---

### 質問フロー

```
Q1「引っ越しの主な理由は何ですか？」
   └─ 仕事・転職のため → 転勤・就職支援 + 移住支援
   └─ 地方に移住したい → Q2（移住先確認）へ
   └─ 家族のため（子育て・介護） → 子育て移住支援へ
   └─ 家賃が払えない → 住宅確保給付金 → 06.失業に誘導
   └─ その他・特に理由なし → Q2へ

Q2「引っ越し先は地方（東京・大阪・名古屋以外）ですか？」
   └─ はい、地方に移住する
   │     → Q3（東京圏からの移住確認）へ
   └─ いいえ、都市内での引っ越し
         → 自治体固有の支援のみ案内

Q3「現在、東京圏（東京・埼玉・千葉・神奈川）に住んでいますか？」
   └─ はい → 移住支援金（最大100万円）対象の可能性大
   └─ いいえ → 移住支援金は対象外（他の移住支援を案内）

Q4「移住先でテレワークを続けますか？または就職予定ですか？」
   └─ テレワーク継続 → 地方移住支援（テレワーク要件確認）
   └─ 移住先で就職 → 移住先自治体の求人マッチング支援
   └─ 起業予定 → 地方起業支援金（最大200万円）へ
```

---

### 提示する制度

| 優先 | 制度名 | 提示条件 |
|---|---|---|
| 🥇 | 移住支援金（最大100万円） | 東京圏→地方 × 就職またはテレワーク |
| 🥈 | 地方起業支援金（最大200万円） | 地方移住 × 起業予定 |
| 🥉 | 子育て移住支援（自治体） | 子あり × 地方移住 |
| ＋ | 住宅確保給付金 | 家賃が払えない状況 |

## 06. 失業

### 入口

```
「仕事を辞めました」「クビになりました」「会社が倒産しました」
「失業保険について知りたい」「無職です、どうしたらいいですか」
```

**AIの第一声**
```
大変な状況ですね。
今使える支援を整理しますので、いくつか確認させてください。
焦らなくて大丈夫です。
```

---

### 質問フロー

```
Q1「仕事を辞めた理由を教えてください」
   └─ 会社の都合（解雇・倒産・リストラ） → 「会社都合」として優遇
   └─ 自分から辞めた（自己都合） → 給付制限3ヶ月があることを説明
   └─ 病気・怪我のため → 傷病手当金の確認へ（Q2A）
   └─ 介護・育児のため → 特定理由離職者として扱われる可能性を説明

Q2「辞める前に、雇用保険に入っていましたか？
   （正社員・パート問わず、週20時間以上働いていた場合は通常入っています）」
   └─ はい（または多分入っていた） → Q3へ
   └─ いいえ（フリーランス・自営業） → Q3'へ
   └─ わからない → ハローワークで確認できることを説明

Q3「雇用保険に入っていた期間はどのくらいですか？」
   └─ 12ヶ月以上 → 基本手当（失業給付）の対象
   └─ 6〜12ヶ月 → 会社都合なら対象になる可能性あり
   └─ 6ヶ月未満 → 対象外の可能性 + 求職者支援制度へ

Q3'（雇用保険なし）
   └─「求職者支援制度」（職業訓練 + 月10万円）の対象確認へ
   └─ 収入が少ない → 住宅確保給付金 + 生活困窮支援へ

Q4「家族（配偶者・子ども）はいますか？」
   └─ 子どもあり（ひとり親） → 児童扶養手当を優先追加
   └─ 配偶者あり → 扶養切り替えの手続きを案内
   └─ 一人暮らし → 単身向けの支援を優先

最終確認Q「住居（家賃）の支払いは今のところ大丈夫ですか？」
   └─ 厳しい → 住宅確保給付金を最優先
   └─ 今は大丈夫 → 念のため住宅確保給付金も紹介
```

---

### 提示する制度

| 優先 | 制度名 | 提示条件 |
|---|---|---|
| 🥇 | 雇用保険（基本手当） | 雇用保険加入 × 12ヶ月以上 |
| 🥈 | 住宅確保給付金 | 家賃支払いが困難 |
| 🥈 | 求職者支援制度 | 雇用保険なし × 職業訓練希望 |
| 🥉 | 高額療養費制度 | 医療費が心配な場合 |
| 🥉 | 国民健康保険の減額制度 | 退職後の保険料が高い場合 |
| ＋ | 児童扶養手当 | ひとり親 |
| ＋ | 生活保護 | 収入・貯蓄が底をつく場合 |

---

### 締めのメッセージ

```
「ハローワークへの登録は、退職翌日から手続きができます。
離職票が届く前でも『仮申請』ができるので、
なるべく早めに行くことをおすすめします。

住宅確保給付金は『収入が急に減った人』が対象なので、
今すぐ申請できる可能性があります。お住まいの市区町村窓口に問い合わせてみてください。」
```

### 入口

```
「転職したい」「転職活動中です」「スキルアップしたい」
「資格を取りながら転職したい」「再就職手当について知りたい」
```

**AIの第一声**
```
転職のタイミングで使える給付金は意外と多くあります。
状況を確認させてください。
```

---

**AIの第一声**


---

### 質問フロー（簡潔に。被災直後は長い質問は禁物）

```
Q1「住居の被害状況を教えてください」
   └─ 全壊・大規模半壊 → 被災者生活再建支援金（最大300万円）を最優先
   └─ 半壊 → 住宅応急修理制度 + 生活再建支援を案内
   └─ 一部損壊 → 災害見舞金（自治体）の確認
   └─ 家は無事だが生活が困難 → 緊急小口資金・生活資金を案内

Q2「現在、住む場所はありますか？」
   └─ 避難所にいる → 応急仮設住宅・みなし仮設（賃貸）の案内を最優先
   └─ 親戚・友人のところ → みなし仮設住宅の申請を案内
   └─ 自宅は使える → 修理・再建支援の案内へ

Q3「家族で亡くなった方はいますか？」（デリケートに）
   └─ はい → 災害弔慰金（最大500万円）を案内
   └─ いいえ → スキップ
```

## Claude API 接続設計

### 現在（ルールベース）→ 将来（Claude API）の比較

```
【現在のアーキテクチャ】
ユーザー入力 → chatFlow.ts（ステートマシン） → 制度マッチング
                     ↑
              事前定義の質問リスト

【Claude API 接続後】
ユーザー入力 → Claude API（自然言語理解） → 制度マッチング
                     ↑
              システムプロンプト + ユーザーコンテキスト
```

---

### システムプロンプト設計

```
あなたは「暮らしAIアシスタント」です。
日本の行政支援制度について、ユーザーの状況を聞いて
使える可能性のある制度を案内するAIです。

【行動原則】
- 専門用語を使わず、わかりやすい言葉で話す
- 共感を示しつつ、必要な情報を簡潔に聞く
- 質問は一度に1つだけ。複数同時に聞かない
- ネガティブな状況（失業・困窮・病気）を責めない
- 「〜の可能性があります」という表現を使い、確定的に言わない
- 制度名より先に「あなたにとってのメリット」を伝える

【ヒアリングの流れ】
1. ユーザーの状況を把握する（1〜2往復）
2. 判断に必要な情報を追加ヒアリング（最大3問）
3. JSON形式でユーザープロフィールを出力
4. 提示する制度のリストを優先度付きで出力

【出力フォーマット（ステップ3以降）】
{
  "userProfile": {
    "age": null or 数値,
    "employment": "employed"|"unemployed"|"self_employed"|"student",
    "familyStructure": "single"|"couple"|"family"|"single_parent",
    "hasChildren": true|false,
    "concerns": ["childcare","housing",...],
    "特記事項": ["難病あり","ひとり親",...]
  },
  "recommendedSubsidies": [
    {
      "id": "national-001",
      "reason": "育休中の会社員のため対象になる可能性があります",
      "priority": 1
    },
    ...
  ],
  "nextMessage": "ユーザーへの次のメッセージ（日本語）"
}

【禁止事項】
- 金額・条件の確定的な断言（「必ずもらえます」等）
- 医療・法律・税務の具体的なアドバイス
- 個人情報（名前・住所・マイナンバー）の入力要求
```

### 段階的な移行計画

```

【Step 2】Claude API を要約生成に活用
```

## カテゴリ別 重要度マップ

```
緊急度（今すぐ支援が必要）
      ↑
高い  │ 12.生活困窮  15.災害  06.失業
      │ 09.病気      10.障害
      │

      │ 05.引越し  07.転職  13.教育
```

**緊急度が高いカテゴリ（生活困窮・災害・失業）では:**
- 質問を最小限にする（最大3問）

- 「今すぐできること」を最初に案内する
